# uni-tools架构设计
uni-tools的目标是提供一个统一的工具集，但是由于不同的工具可能用的技术架构不同

由于前端技术的发展，目前出现了越来越多，越来越复杂的技术架构，如何设计与实现一个跨技术栈的工具集？使得工具可以在同一个应用中可以使用。
目前的方案：
- 使用electron技术实现工具集，同时为子应用提供electron API，用于实现拓展功能，比如文件读取等等。
- 页面渲染，页面渲染采用iframe的形式，每个工具根据自己的技术架构，在iframe中渲染自己的页面。
- 应用加载，每个应用根据自己为用户提供的功能形式，配置app.config.ts/json文件。



- 主应用
-》启动
  - 注册所有子应用的配置文件：说明子应用名称，版本，功能描述，入口文件，显示方式（新窗口，当前窗口）。
=》渲染子应用
  =》 子应用加载
    - 加载子应用的配置文件
  =》 子应用渲染
    - 根据子应用的配置文件，渲染子应用的页面
