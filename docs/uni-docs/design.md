# 架构设计

> 随着数字办公场景的多元化，基于web技术的跨端应用成为前端开发的核心趋势之一。据《2025前端开发者生态报告》显示，Electron框架的使用率已达到42%，广泛应用于代码编辑器、远程协作工具等产品。这类应用的优势在于复用web技术栈，降低跨平台开发成本，但同时也面临着安装包体积大、内存占用高的问题，尤其在低配跨平台开发成本，但同时也面临着安装包体积大、内存占用高的问题，尤其在低配笔记本和嵌入式设备中，用户体验大打折扣，因此对Electron应用的轻量化优化研究具有迫切的实践需求。
>
> 目前，针对Electron优化的研究主要集中在两个方向：一是打包工具的改进，如electron-vite通过整合vite的编译能力，提升了架构效率，但未解决冗余依赖的根本问题；二是运行是的性能调优，如通过关闭不必要的渲染进程降低内存占用，但缺乏系统化的监控和回收机制。总体而言，现有方案大多单一聚焦于编译期或运行时的某一环节，未能实现全链路的协同优化，且针对轻量化工具的定制方案较少。
>
> 针对上述问题，本研究提出一种基于依赖图谱分析的electron全链路轻量化优化方案。该方案首先利用AST解析技术构建应用的依赖图谱，自动剔除未使用的模块和API；其次，设计核心代码+拓展功能的分离打包策略，实现基础功能的快速启动和拓展功能的按需加载；最后，开发运行时内存监控插件，实时检测闲置进程并释放内存碎片。





> 随着前端技术的多元化发展，出现了越来越多的新技术，前端框架有vue、react、angular等；构建工具有vite、webpack、rollup、rolldown、SWC、esbuild等；UI组件库有Element-plus、Ant-design、Material-UI、Native-UI等；css样式工具Tailwind、Unocss、Postcss等；桌面端应用有Electron、Tauri、Neutralinojs；移动端应用有ReactNative、Flutter、Uniapp等。在日常开发中，我们一般会根据项目的需求定制相应的技术栈；在项目中技术栈切换对开发者而言，这会非常消耗他们的精力；那么如何节省这方面的消耗呢，可以通过文档记录累不同技术栈的使用方法、技巧、进阶等；然而要面临这样一个问题，不同技术栈往往不能使用一个项目来记录，比如vitepress只能提供vite+vue技术栈的文档记录、ducura只能提供webpack+react文档的记录。这样不同的技术栈使用不同的项目，采用不同的网址，会严重降低程序员们文档记录的积极性，使用率。那么如何解决呢，只需要将不同技术栈导出文档，及其演示进行规范化，然后在一个主项目中一iframe的形式进行导入，这样就可以聚合不同技术栈的文档，实现不同技术栈文档的聚合。
